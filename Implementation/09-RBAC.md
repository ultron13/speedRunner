# RBAC

Role-based access control protects projects, environments, test data, and execution capabilities.

## Core Roles

- Platform Administrator.
- Performance Engineering Lead.
- Performance Engineer.
- Developer.
- QA Engineer.
- Release Manager.
- Read-Only Viewer.
- Automation Service Account.

## Permission Areas

- Project administration.
- Test catalog management.
- Scenario creation.
- Test execution.
- Production-like environment access.
- Result viewing.
- Result deletion.
- Platform configuration.
- User and role administration.

## Governance Rules

High-volume tests, production-like tests, and restricted environment tests should require approval before execution.
